# Versioning

Schema versions follow semantic versioning: `MAJOR.MINOR.PATCH`.

- **MAJOR**: breaking changes (removed/renamed classes or properties, IRI changes, meaning changes)
- **MINOR**: additive changes (new classes/properties, new optional fields)
- **PATCH**: non-semantic fixes (typos, labels/descriptions, formatting)

Releases are tagged as `v<version>` (for example `v4.0.0`).
